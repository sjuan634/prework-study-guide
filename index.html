HTML

  • The <head> element contains information about the webpage.
  • The <body> element represents the visible content shown to the user.
  • Elements that are nested inside other elements are called child elements. The element that holds all the children elements is called the parent element.
  • <!DOCTYPE html> is a declaration that tells the browser what document to expect. In this case HTML
  • Declaraions are not case-sensitive. HTML elements are case-sensitive.
  • The <html lang="en"> opening tag represents the root, or start of the HTML document. The lang="en" portion is an element attribute.
  • HTML attributes define additional information about an element. They are always defined in the opening tag of an element, never in the closing tag.
  • The <title> element defines the title for the webpage.
  • The <style> element defines te CSS styles associated with a website.
  • The <base> element defines the base URL for the webpage.
  • The <link> element connects an external resource to the HTML document.
  • The <meta> element defines metadata such as character set, description, keywords, author, and viewpoint.
  • The <script> element embeds client-side scripts in the HTML document.
  • <h1>, <h2>, <h3>, <h4>, <h5>, and <h6>, elements represent the level of heading a given text block represents.
  • <p> element represents a paragaph or a block of text.
  • <ul>, <ol>, and <li> elements represent unordered list, ordered list, and list items.
  • <img> element contains information about images that are displayed on the webpage. It can contain various attributes. The src attribute is the most important, it defines the location of the image.
  • <br> element creates a line of empty space, or line break.
  • <a> element stands for anchor, creates links to the same or other webpages.
  • <header> element appears at the top of the page and ussually contains a navigation menu and/or introductory content.
  • <footer> element appears at eh bottom of the page and ussually contains the author, copyright, contact, sitemap, and navigation.
  • <section> elements are semantic elements that define a section in a document.

CSS

  • There are three ways to style a webpage using CSS
    • Inline CSS
    • Internal CSS
    • External CSS
  • Inline CSS is used to style an element directly in the HTML file. Styles apply to specific HTML tag. It is not very flexible for design changes.
  • Internal CSS is used to embed and entire stylesheet directly in the HTML file, using a <style> element in the <head> element. It is more flexible than Inline CSS.
  • External CSS is the most common way to style a webpage. Allows you to keep all CSS rules in a separate file. It makes design changes easier. You use a <link> element in the HTML's <head> element to link the external CSS file.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • Git is a version control system.
  • git clone [SSH value]: clones GitHub repo to local computer
  • git status: checks what branch we are currently on
  • git checkout -b [branch-name]: creates a new branch and switches to it
  • git add -A: adds modifications in the current working branch to the staging area. The -A flag indicates we want to add all changes
  • The staging area informs Git that the changes will be added on the next commit.
  • git commit -m "commit message": creates a commit
  • git pull origin [branch-name]: pulls downs any changes from the branch
  • git push origin [branch-name]: pushes changes to remote branch
